1. 지능 에이전트는 어디에 있을까?

지능 에이전트는 사용자를 대신해 여러 분야에서 자율적으로 작업을 수행합니다. 단순한 컴퓨터 화면을 넘어, 현실 세계의 변화하는 환경을 파악하고 스스로 행동을 결정하죠.

온라인 분야

내 취향을 분석해주는 음악 추천 시스템, 스팸 메일 필터링

산업 분야

스스로 불량품을 찾아내는 공장 로봇, 자율 비행 농업용 드론

서비스 분야

병원에서 환자의 상태를 살피는 돌보미 로봇, 빈 집을 관리하는 스마트홈

2. 에이전트는 어떻게 세상을 이해하고 움직일까?

지능 에이전트가 임무를 수행하려면 사람의 몸과 비슷한 3가지 요소가 필요합니다.

① 센서 (Sensor)

"사람의 눈과 귀"

자신을 둘러싼 외부 환경을 지각하고 데이터를 수집하는 역할을 합니다.
(예: 카메라, 마이크, 온도 감지기)

핵심!

② 지능 (소프트웨어)

"사람의 두뇌"

센서로 수집한 정보를 분석하고, 어떻게 행동할지 학습하고 추론하여 결정합니다.

③ 액추에이터 (Actuator)

"사람의 팔과 다리"

지능의 명령을 받아 실제로 환경에 물리적인 변화를 일으키는 장치입니다.
(예: 로봇 팔, 스피커, 바퀴, 화면 표시)

헷갈리지 마세요! (오개념 방지)

"자동문도 사람이 오면 열리는데, 지능 에이전트인가요?"
자동문은 단순한 '센서'와 모터(액추에이터)만 있습니다. 하지만 상황을 보고 스스로 판단(누구인지, 열어줘야 할지 말지)하는 '지능(소프트웨어)' 요소가 없기 때문에 단순한 기계일 뿐, 지능 에이전트는 아닙니다.

Step-by-step Design

3. 스마트홈 경비원 만들기

집을 지키는 '스마트홈 에이전트'를 만든다고 상상해 봅시다. 교과서에 나온 설계 과정을 단계별로 볼까요?

  • 1
    목표 설정 집 내부가 안전한지 확인한다.
  • 2
    인식 (센서) 가족 외에 다른 사람의 침입이 없는지 카메라로 관찰한다.
  • 3
    학습 및 추론 (지능) 카메라에 찍힌 사람이 무단 침입자인지, 이상 행동을 하는지 분석한다.
  • 4
    행동 (액추에이터) 침입이 확인되면 경고음을 울리거나 보안 업체에 연락한다.
smart_home.c 

// 스마트홈 보안 에이전트의 작동 흐름

void runSecurityAgent() {
    while (true) {
        // 1. 인식 (Sensor)
        Image currentView = cameraSensor.captureImage();

        // 2. 추론 (Intelligence)
        Person detectedPerson = aiEngine.analyzeFace(currentView);

        if (detectedPerson == UNKNOWN_PERSON) {
            // 가족이 아닌 낯선 사람으로 추론됨!

            // 3. 행동 (Actuator)
            speakerActuator.soundAlarm();  // 경고음 울리기
            networkModule.callPolice();  // 보안업체 연락

        } else if (detectedPerson == FAMILY_MEMBER) {
            // 가족이면 문을 열어준다 (행동)
            doorActuator.unlock();
        }
    }
}
