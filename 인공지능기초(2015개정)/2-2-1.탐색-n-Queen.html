백트래킹 (Backtracking) 이란?

문제를 해결하기 위해 모든 가능성을 탐색하다가, 현재 경로가 정답이 될 수 없다고 판단되면 (유망하지 않으면) 즉시 이전 단계로 되돌아가서(Backtrack) 다른 경로를 탐색하는 알고리즘 기법입니다.

💡 핵심 포인트

  • 상태 공간 트리(State Space Tree): 문제 해결 과정을 트리 형태로 구성하여 탐색합니다.
  • 가지치기(Pruning): 정답이 될 가능성이 없는 노드는 더 이상 탐색하지 않고 잘라내어 탐색 시간을 대폭 줄입니다.

준비 단계: N-Queen 규칙 알아보기

체스 게임의 '여왕(Queen)'은 가장 강력한 말입니다. N-Queen 퍼즐의 3가지 핵심 규칙을 살펴볼까요?

1. 기본 구성

가로 N칸, 세로 N칸의 체스판에 N개의 퀸(Queen)을 배치해야 합니다. (예: 4x4 판에는 4개의 퀸 배치)

2. 퀸의 파워

퀸은 가로, 세로, 그리고 대각선 방향으로 칸 수에 제한 없이 끝까지 이동하며 공격할 수 있습니다.

3. 최종 목표

어떤 퀸들도 서로 공격할 수 없도록, 즉 같은 행, 열, 대각선에 퀸이 겹치지 않게 모두 안전하게 배치하세요.

마무리 단계: 코드로 보는 원리

코드로 보는 DFS와 백트래킹 원리

오른쪽 화면에 그려진 트리 구조는 아래 파이썬 코드의 재귀 호출(Recursive Call) 흐름과 완전히 일치합니다.
함수가 자기 자신을 호출하면 트리가 아래로(자식 노드) 깊어지고, 가지치기되어 `return`을 만나면 위로(부모 노드) 되돌아옵니다.

Python 
# DFS(깊이 우선 탐색)를 이용한 N-Queen 백트래킹
def solve_n_queens(row, n, state):
    if row == n:
        return True # 트리 끝단 도달! 정답을 찾음

    for col in range(n):
        # 1. 시도 (Try) 및 가지치기(Pruning) 검사
        if is_safe(row, col, state):
            state[row] = col  # 안전하다면 퀸을 배치 (Place)

            # 2. 다음 줄(depth) 탐색을 위해 재귀 호출 (자식 노드로 이동)
            if solve_n_queens(row + 1, n, state):
                return True

            # 3. 백트래킹 (Backtracking)
            # 자식 노드에서 답을 못 찾고 돌아왔으므로 상태를 복구하고 다른 칸 시도
            state[row] = -1

    return False # 이 노드의 모든 자식 경로에 답이 없음
