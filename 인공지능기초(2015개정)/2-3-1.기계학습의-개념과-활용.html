Chapter 02.

기계학습의 개념과 활용

유튜브나 넷플릭스는 어떻게 내가 좋아할 만한 영상을 쏙쏙 골라줄까요?
데이터를 통해 스스로 규칙을 깨우치는 '기계학습(Machine Learning)'의 비밀을 알아봅시다!

인공지능 기초
기계학습이란? 추천 시스템 원리 코드 예제 확인질문

컴퓨터에게 가르치는 두 가지 방법

인공지능을 만드는 방법은 크게 '규칙 기반(Rule-based)'과 '학습 기반(Learning-based)'으로 나눌 수 있습니다. 기계학습(Machine Learning)은 바로 이 '학습 기반'의 대표적인 방법입니다.

전통적인 방식 (규칙 기반)

사람이 컴퓨터에게 모든 규칙을 직접 하나하나 알려주는 방식입니다.

예시:
"다리가 4개고, 꼬리가 있고, '멍멍' 소리를 내면 그것은 '강아지'라고 판단해!"
👉 단점: 예외 상황(다리가 3개인 강아지)이 생기면 컴퓨터가 고장 납니다.

기계학습 (학습 기반)

컴퓨터에게 수많은 데이터를 주고, 스스로 규칙(패턴)을 찾게 하는 방식입니다.

예시:
강아지 사진 1만 장과 고양이 사진 1만 장을 보여주며 "이게 강아지고, 저게 고양이야"라고 알려줍니다.
👉 장점: 컴퓨터가 스스로 생김새의 특징을 파악하여 새로운 사진도 잘 분류합니다!

내 맘을 읽는 마법? '추천 시스템'의 원리

기계학습이 가장 활발하게 쓰이는 곳이 바로 유튜브나 넷플릭스 같은 '추천 서비스'입니다. 수백만 개의 영상 중에서 어떻게 내가 좋아할 만한 영상을 골라낼까요? 크게 두 가지 방법이 있습니다.

방법 1 콘텐츠 기반 필터링 (Content-based Filtering)

"영상의 내용(특징)을 분석해서 비슷한 걸 추천해 줄게!"

사용자가 과거에 시청한 영상의 특징(장르, 키워드, 등장인물 등)을 파악하여, 그와 비슷한 특징을 가진 다른 영상을 추천합니다.


본 영상(게임)

추천(다른 게임)

방법 2 협업 기반 필터링 (Collaborative Filtering)

"너랑 취향이 비슷한 다른 사람들이 본 걸 추천해 줄게!"

나와 비슷한 콘텐츠를 소비한(비슷한 취향을 가진) 다른 사용자들을 그룹으로 묶은 뒤, 그 사람들이 많이 본 새로운 영상을 나에게도 추천합니다.


나 (A, B 시청)
취향 비슷

타인 (A, B, C 시청)
C
추천
💡 실제 유튜브나 넷플릭스는 이 두 가지 방법과 다른 여러 가지 기술을 혼합하여(하이브리드) 더 정확한 추천을 해냅니다.

코드로 보는 '협업 필터링'의 기본 원리

"나와 취향이 비슷한 사람을 찾는다"는 것은 컴퓨터가 어떻게 계산할까요? 파이썬(Python) 형태의 가짜 코드(의사코드)로 그 논리를 쉽게 살펴봅시다.

recommendation_logic.py 

# 1. 사용자들의 시청 기록 데이터 수집 (어떤 채널을 구독/시청했는지)
user_me = ["게임", "음악", "요리"]
user_friend_A = ["뷰티", "패션", "브이로그"]
user_friend_B = ["게임", "음악", "IT기기 리뷰"]  # 나와 취향이 비슷함!

# 2. 취향 유사도(Similarity) 계산 함수
def check_similarity(user1, user2):
    # 겹치는 관심사의 개수를 셉니다.
    common_interests = set(user1) & set(user2)
    return len(common_interests)

# 3. 나와 가장 비슷한 사람 찾기
score_A = check_similarity(user_me, user_friend_A) # 겹치는 것 0개
score_B = check_similarity(user_me, user_friend_B) # '게임', '음악' 겹침 (2개)

if score_B > score_A:
    print("친구 B와 취향이 비슷하군요!")
    # 4. B는 봤지만 나는 안 본 영상 추천하기
    print("친구 B가 즐겨보는 'IT기기 리뷰' 영상을 추천합니다!")

잠깐! 오해하지 마세요 (오개념 방지)

"기계학습 인공지능은 처음부터 똑똑해서 다 알아맞힌다?"

아닙니다! 인공지능은 태어날 때(?) 아무것도 모르는 백지상태입니다. 기계가 학습하기 위해서는 엄청난 양의 '데이터(Data)'가 필수적입니다. 우리가 교과서로 공부하듯, 인공지능도 양질의 데이터를 많이 보며 학습해야만 정확한 결과를 낼 수 있습니다. 쓰레기 데이터를 넣으면 쓰레기 결과가 나옵니다. (Garbage In, Garbage Out)

📌 오늘의 핵심 정리

  • 기계학습은 사람이 규칙을 주지 않고, 기계가 데이터에서 스스로 패턴을 찾는 학습 기반 인공지능이다.
  • 콘텐츠 기반 필터링: 영상 자체의 특징(장르 등)이 비슷한 것을 추천한다.
  • 협업 기반 필터링: 나와 시청 기록(취향)이 비슷한 다른 사람의 영상을 추천한다.
