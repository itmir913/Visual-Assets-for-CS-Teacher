Chapter 2. C 프로그래밍의 기초

1. 변수와 자료형

데이터를 메모리에 저장하는 변수의 개념을 익히고, 데이터 크기에 따른 효율적인 자료형 선택 방법을 배웁니다.

Think & Objective

"컴퓨터는 숫자와 문자를 내부적으로 어떻게 구분하여 저장할까요?"

  • 변수의 개념을 이해하고 적절한 자료형으로 변수를 선언할 수 있다.
  • 자료형의 특성과 메모리 크기를 이해하고 효율적인 코딩을 할 수 있다.

C 프로그램의 첫걸음: 기본 뼈대 이해하기

처음 C언어 코드를 보면 마치 암호문 같아 보일 수 있습니다. 하지만 겁먹을 필요 없습니다! C 프로그램은 항상 정해진 기본 뼈대(구조) 안에서 작성됩니다. 아래의 가장 기초적인 "Hello, World" 출력 코드를 보며, 각 줄이 어떤 역할을 하는지 해부해 볼까요?

예제 기본 뼈대 코드
63.c 
#include <stdio.h>

int main()
{
	printf("Hello, World\n");
	return 0;
}

#include <stdio.h>

"도구 상자 가져오기!" 화면에 글자를 보여주거나 키보드 입력을 받기 위해 필요한 기본 도구들을 프로그램에 포함시키는 주문입니다.

int main() { ... }

"프로그램의 현관문!" 컴퓨터가 프로그램을 실행할 때 가장 먼저 찾아서 들어가는 시작점입니다. 모든 코드는 이 { } 중괄호 안에 들어갑니다.

printf("Hello, World\n");

"화면에 출력해 줘!" 따옴표 안의 글자를 화면에 보여주는 명령어입니다. 끝에 있는 \n은 '엔터(줄바꿈)'를 의미합니다.

return 0;

"무사히 잘 끝났습니다!" 프로그램이 아무런 에러 없이 성공적으로 작업을 마쳤다는 것을 컴퓨터에게 알려주며 종료하는 의미입니다.

변수 이름 지어주기: 맘대로 지어도 될까요?

변수는 데이터를 담는 상자입니다. 상자 안에 무엇이 들었는지 쉽게 알 수 있도록 이름표를 붙여주어야 하는데, 컴퓨터가 인식할 수 있도록 몇 가지 지켜야 할 '문법 규칙'이 있습니다. 마치 게임에서 새로운 아이디(닉네임)를 만들 때 특수문자나 띄어쓰기를 막아두는 것과 똑같습니다!

변수 이름 짓기 4가지 절대 규칙

  • 영문자, 숫자, 밑줄(_)만 사용할 수 있습니다. (한글이나 ?, !, @ 같은 특수문자는 안 돼요!)
  • 숫자로 시작할 수 없습니다. (예: 1st_score 금지 ❌ → score_1st 가능 ⭕)
  • 띄어쓰기를 할 수 없습니다. 단어를 연결하고 싶다면 밑줄을 쓰세요! (예: my score ❌ → my_score ⭕)
  • C언어가 이미 사용 중인 예약어(int, return 등)는 쓸 수 없습니다.
비교 실습 O / X 로 알아보는 변수 이름
naming_rules.c 
#include <stdio.h>

int main() {
    // ⭕ 올바른 변수 이름 (권장)
    int age = 17;
    int my_score = 100;    // 밑줄로 단어 연결
    int player1_hp = 50;   // 중간이나 끝에 숫자 사용

    // ❌ 에러가 발생하는 잘못된 변수 이름
    // int 1st_player = 1; // 숫자로 시작 불가!
    // int my score = 100; // 띄어쓰기 불가!
    // int user@id = 5;    // 특수문자 불가!
    // int return = 0;     // 예약어 사용 불가!

    return 0;
}

자료형(Data Type)

숫자 데이터

숫자는 소수점 유무에 따라 정수와 실수로 나뉩니다.

int 정수 (4 bytes)

소수점이 없는 숫자 (-1, 0, 100 등)를 담습니다.

int age = 17;
float / double 실수 (4/8 bytes)

소수점이 있는 숫자 (3.14, 175.5 등)를 담습니다.

double pi = 3.141592;

문자 데이터

컴퓨터는 문자 하나하나를 저장할 때 숫자로 저장합니다.

char 문자 하나 (1 byte)

작은따옴표(' ')를 사용해 글자 하나만 담습니다.

char grade = 'A';
문법 주의!
  • 문자는 'A' (작은따옴표)
  • 문자열은 "Hello" (큰따옴표)
  • C언어에서 `char`는 딱 한 글자만 저장합니다.

C언어만의 독특한 출력 방식: %d, %c는 대체 뭘까?

C언어 코드를 보다 보면 printf("결과: %d", a); 처럼 %d, %c 같은 기호가 계속 등장합니다. 초보자분들이 가장 헷갈려 하는 부분 중 하나인데요! C언어에서는 변수의 값을 화면에 보여줄 때, 어떤 형태(숫자, 문자 등)로 보여줄지 미리 자리를 찜해두어야 합니다. 이를 서식 지정자라고 부릅니다.

식당의 '예약석 팻말' 비유

문장 안에 변수의 값이 들어갈 자리를 미리 비워두고 팻말을 세워둔다고 생각해보세요.

  • %d : "여기는 정수(Decimal) 손님이 앉을 자리입니다!"
  • %c : "여기는 문자(Character) 손님이 앉을 자리입니다!"
  • %f : "여기는 실수(Float) 손님이 앉을 자리입니다!"

따옴표 안의 %d 위치에, 따옴표 뒤에 있는 변수 a의 값이 쏙 들어가게 됩니다!

개념 확인 예약석에 변수 값 앉히기
format_specifier.c 
#include <stdio.h>

int main() {
    int age = 17;
    char grade = 'A';

    // %d 자리에 age(17)가, %c 자리에 grade('A')가 들어갑니다.
    // 여러 개를 출력할 때는 쉼표(,)로 순서대로 연결해 줍니다.
    printf("제 나이는 %d살이고, 프로그래밍 성적은 %c입니다.\n", age, grade);

    return 0;
}

자료형과 sizeof 연산자

숫자와 문자를 처리하기 위해서는 컴퓨터 내부의 기억 장치에 저장시켜야 하는데, 효율적인 메모리 공간 활용을 위해서 C 언어에서는 기본 자료형을 제공합니다.

실습 자료형 크기 측정
65-1.c 
#include <stdio.h>
int main()
{
	printf("char: %d\n", sizeof(char));
	printf("int: %d\n", sizeof(int));
	printf("double: %d\n", sizeof(double));
	return 0;
}
실습 연산 시 크기 확장
65-2.c 
#include <stdio.h>
int main()
{
	char num1 = 1, num2 = 2;
	printf("%d", sizeof(num1 + num2));
	return 0;
}

잠깐! 1바이트 + 1바이트가 4바이트라고요? (정수 승격)

앞선 실습(65-2.c)에서 1바이트인 char형 변수 두 개를 더했는데, 크기가 4바이트로 측정되어 놀라셨나요? 이는 C언어가 가진 '정수 승격(Integer Promotion)'이라는 독특한 특징 때문입니다. 컴퓨터의 뇌(CPU)가 가장 편안하게 계산할 수 있는 최적의 상태로 데이터를 자동 변환하는 마법입니다.

CPU의 작업 효율성

컴퓨터의 CPU는 기본적으로 4바이트(int) 단위로 데이터를 처리할 때 가장 빠릅니다. 그래서 작은 크기의 데이터(char, short)끼리 덧셈, 뺄셈 같은 연산을 시도하면 다음과 같은 일이 발생합니다.

  • 연산을 시작하기 직전, 작은 데이터들을 임시로 가장 효율적인 크기인 int(4바이트)로 늘립니다.
  • 공간을 넉넉하게 늘려서 계산하므로, 연산 도중 값이 넘쳐서(오버플로우) 데이터가 손실되는 것을 방지하는 안전장치 역할도 합니다.
num1 char (1B)
num2 char (1B)
CPU 연산 시 자동 변환 int (4 Bytes)

변수(Variable)의 이해

프로그래밍에서 변수란 데이터를 저장할 수 있는 메모리 상의 이름 붙은 공간입니다. 컴퓨터에게 "내가 'age'라는 상자를 만들 테니까, 여기엔 숫자를 담아줘!"라고 부탁하는 것과 같습니다.

선언

변수의 이름과 종류(자료형)를 정해주는 단계입니다.

할당/초기화

대입 연산자(=)를 사용해 값을 상자에 넣는 단계입니다.

참조/변경

변수에 담긴 값을 읽어오거나 새로운 값으로 바꿉니다.

Step 1

가장 기초: 변수 만들고 출력하기

#include <stdio.h>

int main() {
    // 1. 선언: 정수(int)를 담을 a라는 상자를 준비합니다.
    int a;

    // 2. 초기화: a 상자에 10이라는 숫자를 넣습니다.
    a = 10;

    // 3. 출력: %d는 정수를 출력하라는 약속된 기호입니다.
    printf("상자 a의 값: %d\n", a);

    return 0;
}
Step 2

응용: 변수의 값 바꾸기

#include <stdio.h>

int main() {
    int score = 80; // 선언과 동시에 값을 넣을 수 있습니다.
    printf("처음 점수: %d\n", score);

    score = 100;    // 변수는 값이 변할 수 있어서 '변수'입니다.
    printf("수정된 점수: %d\n", score);

    return 0;
}

변수의 용량 초과! 오버플로우(Overflow)란?

각 자료형은 담을 수 있는 숫자의 한계점(최댓값과 최솟값)이 정해져 있습니다. 만약 최댓값 상태에서 1을 더 추가하면 어떻게 될까요? 에러가 발생할 것 같지만, C언어에서는 아주 독특한 현상이 일어납니다.

자동차 계기판의 원리

아날로그 자동차 주행거리 계기판을 떠올려보세요. 계기판의 최대 표시가 99,999km일 때, 여기서 1km를 더 달리게 되면 어떻게 될까요?

  • 100,000km를 표시할 공간이 없기 때문에 계기판은 한 바퀴를 돌아 00,000km로 초기화됩니다.
  • C언어의 변수도 똑같습니다. 최댓값을 넘어가는 순간 공간이 부족해지면서, 범위의 가장 작은 최솟값(음수)으로 둥글게 돌아가 버립니다. 이를 오버플로우(Overflow)라고 부릅니다.

최댓값 + 1 → 최솟값으로 순환!

개념 확인 char형으로 확인하는 순환 현상
overflow_concept.c 
#include <stdio.h>
int main() {
    // char는 -128 부터 127 까지 저장할 수 있습니다.
    char max_val = 127;

    // 127에 1을 더하면 128이 아니라 최솟값으로 돌아갑니다.
    max_val = max_val + 1;

    printf("결과: %d\n", max_val); // 결과: -128
    return 0;
}

잠깐! 변수 값을 바꿀 때의 주의점 (덮어쓰기)

본격적으로 두 변수의 값을 서로 교환하기 전에 꼭 알아야 할 특징이 있습니다. 변수에 새로운 값을 넣으면, 기존에 있던 값은 지워지고 완전히 덮어쓰기가 됩니다. 이 특징 때문에 단순히 두 변수를 서로 대입하는 것만으로는 값을 맞바꿀 수 없습니다.

실패 예제 단순 대입으로 교환 시도하기
swap_fail.c 
#include <stdio.h>
int main() {
	int a = 10;
	int b = 20;

	// a에 b의 값을 넣습니다. (a의 10은 사라지고 20이 됩니다)
	a = b;
	// b에 a의 값을 넣습니다. (a는 이미 20이므로 b도 20이 됩니다)
	b = a;

	printf("a: %d, b: %d\n", a, b); // 둘 다 20이 출력됨!
	return 0;
}

컵 안의 내용물 바꾸기 비유

a 컵에는 주스가, b 컵에는 우유가 들어있다고 상상해 보세요.

  • b 컵의 우유를 a 컵에 바로 부어버리면 주스와 섞이거나 넘쳐서(덮어쓰기) 주스는 사라집니다.
  • 내용물을 안전하게 바꾸려면 어떻게 해야 할까요? 바로 아무것도 들어있지 않은 빈 컵(temp)이 하나 더 필요합니다!

이 원리를 바탕으로 다음 실습의 'temp'를 살펴보세요.
