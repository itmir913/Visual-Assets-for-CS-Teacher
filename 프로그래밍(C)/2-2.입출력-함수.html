Chapter 2. C 프로그래밍의 기초

2. 입출력 함수

컴퓨터와 사용자 사이의 소통 창구인 표준 입출력 함수를 배우고, 데이터를 원하는 형식으로 제어하는 방법을 익힙니다.

Think & Objective

"형식을 지정하여 입력과 출력을 하는 함수는 어떻게 사용할까?"

  • 입출력 함수의 기능을 이해하고 설명할 수 있다.
  • 형식을 지정하여 데이터를 입력하고 출력할 수 있다.

문자 및 문자열 출력 (putchar, puts)

C언어에서 화면에 결과를 보여주는 대표적인 함수는 printf입니다. 하지만 오직 '문자'나 '문장'만 출력할 때는 더 가볍고 쓰기 편한 전용 도구들이 있습니다. 마치 상황에 맞춰 작은 드라이버와 전동 드릴을 골라 쓰는 것과 같죠! 두 가지 도구의 차이점을 알아볼까요?

putchar : "딱 한 글자만!"

오직 문자 1개(작은따옴표 ' ')만 출력하는 가벼운 도구입니다. 글자를 출력한 뒤 다음 줄로 넘어가지 않고 그 자리에 가만히 멈춰 있습니다. (줄바꿈을 하려면 putchar('\n');을 따로 써주어야 해요!)

puts : "문장 전체 + 자동 줄바꿈!"

여러 글자로 이루어진 문장(큰따옴표 " ")을 한 번에 출력하는 도구입니다. 가장 편한 점은 문장을 통째로 출력한 뒤 알아서 엔터(줄바꿈)를 쳐준다는 것입니다!

개념 확인 putchar vs puts 차이점 확인하기
print_basic.c 
#include <stdio.h>

int main() {
    // 1. putchar 사용 (작은따옴표)
    putchar('H');
    putchar('i');
    putchar('\n'); // 직접 줄바꿈 기호를 넣어줘야 함

    // 2. puts 사용 (큰따옴표)
    // 출력 후 '알아서' 줄바꿈이 됩니다!
    puts("Hello!");
    puts("C언어 재미있네요.");

    return 0;
}

컴퓨터는 문자를 어떻게 기억할까? (아스키코드)

앞선 실습에서 putchar(105); 처럼 숫자를 넣었는데 소문자 'i'가 출력되어 당황하셨나요? 사실 컴퓨터는 글자를 있는 그대로 기억하지 못합니다. 대신 "A는 65번, B는 66번, a는 97번으로 부르자!"라는 전 세계적인 약속을 만들었는데, 이를 아스키(ASCII) 코드라고 부릅니다.

마법의 숫자, 아스키 코드!

아스키 코드를 외울 필요는 전혀 없습니다! 문자를 표현하기 위해 컴퓨터 내부적으로 문자를 숫자로 표현한다는 점만 기억하세요!

  • 'A' 65 부터 시작
  • 'a' 97 부터 시작 (대문자보다 32가 큼!)
  • '0' 숫자 0은 48 (값 0과는 다름 주의)
개념 확인 대소문자 변환의 비밀
ascii_secret.c 
#include <stdio.h>
int main() {
    char ch = 'a'; // 소문자 a (97)

    // 97에서 32를 빼면 65('A')가 됩니다!
    // 이 원리가 나중에 '암호화 프로그램' 실습에 쓰입니다.
    putchar(ch - 32);

    return 0;
}

표준 출력 함수: printf()

printf() 함수는 "print formatted"의 줄임말로, 데이터를 특정한 형식에 맞춰 화면에 출력합니다. 단순히 글자만 찍는 것이 아니라, 변수의 값을 문장 사이에 끼워 넣을 수 있습니다.

문법 핵심 가이드

  • 1. 큰따옴표(" ") 안에 출력 내용을 적습니다.
  • 2. \n 기호로 줄바꿈을 합니다.
  • 3. %d, %f 등 서식 지정자를 사용합니다.
Step 1

단순 문자열 출력

#include <stdio.h>

int main() {
    printf("안녕하세요!\n");
    printf("C언어 공부 시작!");
    return 0;
}

표준 입력 함수: scanf()

scanf() 함수는 사용자(키보드)로부터 데이터를 입력받아 변수에 저장하는 역할을 합니다. scanf()는 "Scan Formatted"의 약자로, 입력받을 데이터의 형식을 printf()처럼 서식 지정자(%d, %f 등)로 미리 지정해 주어야 합니다.

초보자가 가장 많이 하는 실수!

scanf를 사용할 때 변수 이름 앞에 반드시 & (앰퍼샌드)를 붙여야 합니다!

scanf("%d", &age);
scanf("%d", age);

& 기호는 해당 변수의 "메모리 주소"를 알려주는 역할입니다. 택배 기사님(scanf)에게 물건을 넣을 집 주소를 정확히 알려줘야 하는 것과 같습니다.

Step 1 정수 하나 입력받기
scanf_basic.c 
#include <stdio.h>

int main() {
    int num;
    printf("숫자를 입력하고 엔터를 치세요: ");

    // 입력 대기 상태가 됩니다. (& 기호 주의!)
    scanf("%d", &num);

    printf("당신이 입력한 숫자는 %d입니다.\n", num);
    return 0;
}
입력창이 안 나오고 프로그램이 그냥 끝나버렸나요?

scanf를 여러 번 연달아 쓰다 보면, 두 번째 입력을 받지 않고 프로그램이 휙 넘어가 버리는 에러를 자주 겪게 됩니다.
컴퓨터 고장이 아니니 당황하지 마세요! 👉 [C언어 최대의 함정: 엔터키의 배신] 파트를 읽어보고 스스로 문제를 해결해 봅시다!

문자 입력 (getchar)

이제 컴퓨터가 일방적으로 말하는 것을 넘어, 사용자(여러분)의 키보드 입력을 기다리게 만드는 첫 번째 함수를 배웁니다! getchar()는 이름 그대로 딱 하나의 문자(Character)만 가져오는(Get) 기능을 합니다. 키보드를 치고 엔터키를 누르는 순간 첫 번째 글자 하나만 변수 안으로 쏙 들어갑니다.

데이터 입력 (gets, scanf)

한 글자가 아니라 긴 단어, 문장, 혹은 숫자를 통째로 입력받고 싶을 때 사용하는 핵심 함수들입니다. 특히 scanf는 printf와 짝꿍으로, 서식 지정자를 이용해 "어떤 형태의 데이터를 받을지"를 미리 지정해 줍니다.

gets() 함수 실습 전 주의사항

최신 프로그래밍 환경(Visual Studio 최신 버전 등)에서는 보안 문제(버퍼 오버플로우)로 인해 gets 대신 fgets라는 더 안전한 함수를 쓰도록 강력히 권장하고 있습니다.

출력물을 예쁜 표처럼 정렬하기 (%5s, %3d)

여러 코드를 접하다보면, %d 중간에 갑자기 숫자가 들어간 %5s, %3d 같은 기호가 등장합니다. 이것은 에러가 아닙니다! 모니터 화면에 데이터를 출력할 때, 미리 정해진 칸 수만큼 공간을 확보하여 깔끔하게 줄을 맞추는 마법의 주문입니다.

자리수 지정의 원리

  • %5s 미리 5칸의 빈 공간을 만들고, 글자를 오른쪽에 바짝 붙여서 출력합니다.
  • %3d 미리 3칸의 빈 공간을 만들고, 숫자를 오른쪽으로 정렬하여 출력합니다. (숫자의 자릿수를 맞출 때 아주 유용해요!)

printf("[%5s]", "G5");

]

printf("[%3d]", 45);

]

앞에 남는 공간은 스페이스바(공백)로 채워집니다.

심화 과정 입력이 자꾸 무시된다면 클릭하세요!

C언어 최대의 함정: 엔터키의 배신 (입력 버퍼)

다음 실습 7-2인 '사용자 프로필 입력'을 하기 전에 반드시 알아야 할 C언어의 큰 특징이 있습니다. 나이를 입력하고 [Enter]를 치면, 숫자는 변수에 쏙 들어가지만 친 엔터키(\n) 자체는 컴퓨터의 대기실(버퍼)에 몰래 남아있게 됩니다. 그래서 다음번 혈액형(문자 하나)을 입력받을 때, 컴퓨터가 이 남은 엔터키를 혈액형 문자로 착각해버리고 홀연히 넘어가 버리는 치명적인 에러가 발생합니다.

찌꺼기 처리하는 마법의 '공백'

이 문제를 해결하는 방법은 아주 간단하면서도 독특합니다. scanf 안의 %c 앞에 공백(스페이스)을 하나 딱 넣어주면 됩니다!

scanf("%c", &blood); ❌ 실패

scanf(" %c", &blood); ⭕ 성공!

%c 앞의 띄어쓰기는 컴퓨터에게 "대기실에 남아있는 엔터, 스페이스바 같은 쓸데없는 찌꺼기들은 다 무시하고, 진짜 의미 있는 글자 하나만 가져와!" 라고 명령하는 약속입니다.

버퍼(대기실)의 상태
1단계 나이 17 입력 후 엔터(\n)
1
7
\n

↑ 17은 가져갔지만 \n(엔터)이 남음!

심화 데이터 처리 (칸 수 맞추기, 진법 변환)

데이터를 좀 더 정교하게 다루는 심화 과정입니다. 앞선 보충 설명에서 배운 칸 수 맞추기(%5s, %3d)와 입력 버퍼 찌꺼기 무시하기(" %c") 원리가 본격적으로 등장합니다. 또한, 우리가 쓰는 10진수뿐만 아니라 컴퓨터가 좋아하는 8진수(%o), 16진수(%x)로 숫자를 변환해 보는 흥미로운 실습도 포함되어 있습니다.

실습 7-1
정렬된 표 출력 (서식 지정)
82.c 
#include <stdio.h>
int main()
{
	printf("--------------------------------\n");
	printf("%s\t%s\t%s\t%s\n", "상품명", "총수량", "재고", "가격");
	printf("--------------------------------\n");
	printf("%5s \t %3d \t %3d \t %7d\n", "G5",100,45,965000);
