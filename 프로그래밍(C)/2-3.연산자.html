Chapter 2. C 프로그래밍의 기초

3. 연산자 (Operator)

데이터를 가공하고 계산하기 위한 연산자의 종류를 배우고, 우선순위와 결합 법칙을 통해 복합적인 수식을 이해합니다.

Think & Objective

"컴퓨터는 연산의 순서를 어떻게 결정할까요?"

  • 연산, 비교, 논리 등의 여러 종류의 연산자의 의미를 알고 프로그래밍에 적절하게 사용할 수 있다.
  • 다양한 연산자를 활용하여 프로그래밍 할 수 있다.

산술 연산자란?

가장 기본이 되는 연산자로, 수학의 사칙연산과 유사합니다. 나머지 연산(%)은 프로그래밍에서 홀짝 구분이나 배수 판별에 자주 쓰이니 꼭 기억하세요!

문법 포인트: 나누기와 나머지

  • / (나누기): 정수끼리 나누면 결과도 정수입니다. (예: 5 / 2 = 2)
  • % (나머지): 나눈 후 남은 값을 구합니다. (예: 5 % 2 = 1)

우선순위 주의

곱셈(*), 나눗셈(/), 나머지(%) 연산이 덧셈(+), 뺄셈(-)보다 먼저 계산됩니다. 우선순위를 바꾸려면 괄호( )를 사용하세요.

Step 1

기초 사칙연산

#include <stdio.h>

int main() {
    int a = 10, b = 3;

    printf("덧셈: %d + %d = %d\n", a, b, a + b);    // 13
    printf("뺄셈: %d - %d = %d\n", a, b, a - b);    // 7
    printf("곱셈: %d * %d = %d\n", a, b, a * b);    // 30
    printf("나눗셈: %d / %d = %d\n", a, b, a / b);  // 3 (소수점 버림)
    printf("나머지: %d %% %d = %d\n", a, b, a % b); // 1 (나눈 나머지)

    return 0;
}

대입 및 증감 연산자

단순히 값을 넣는 = 외에도 기존 값에 연산을 더하는 복합 대입 연산자가 있습니다. 또한 ++나 --를 사용하면 값을 1씩 아주 간편하게 늘리거나 줄일 수 있습니다.

복합 대입 연산자 예시
a += 1
(a = a + 1 과 같음)
a *= 2
(a = a * 2 와 같음)
전위 vs 후위 (어려움 주의!)
  • ++a (전위): 1을 먼저 더하고 문장을 실행합니다.
  • a++ (후위): 문장을 실행한 후 나중에 1을 더합니다.
Step 2

증감 및 복합 연산 실습

#include <stdio.h>

int main() {
    int score = 10;

    score += 5; // score = 10 + 5
    printf("복합 대입 결과: %d\n", score); // 15

    score++;    // score = 15 + 1
    printf("1 증가 결과: %d\n", score); // 16

    score--;    // score = 16 - 1
    printf("1 감소 결과: %d\n", score); // 15

    return 0;
}

비교 및 논리 연산자

조건문에서 "두 값이 같은가?" 혹은 "A이고 B인가?"와 같은 질문을 던질 때 사용합니다. C언어에서 참은 1, 거짓은 0으로 표현된다는 점이 매우 중요합니다.

비교 연산자 (결과는 0 또는 1)

== 양쪽 값이 같다
!= 양쪽 값이 다르다
>, <= 크다, 작거나 같다
Step 3

참/거짓 출력하기

#include <stdio.h>

int main() {
    int a = 10, b = 20;

    printf("a == b 결과: %d\n", a == b); // 0
    printf("a < b 결과: %d\n", a < b);   // 1
    printf("a != b 결과: %d\n", a != b); // 1

    return 0;
}

논리 연산자 (조건 결합)

&& (AND) 둘 다 참일 때만 참
|| (OR) 둘 중 하나라도 참이면 참
! (NOT) 참/거짓을 뒤집음
Step 4

복합 조건문 이해

#include <stdio.h>

int main() {
    int a = 10;
    // a가 0보다 크고(&&) 20보다 작은가?
    printf("0 < a < 20: %d\n", (a > 0) && (a < 20)); // 1
    return 0;
}

산술 및 대입 연산자

워밍업: 사칙연산과 나머지(%) 연산자

교과서 예제를 보기 전에, 아주 간단한 숫자로 먼저 원리를 파악해 봅시다.
C언어에서 +, -, *, /는 우리가 아는 수학과 같지만, % (나머지 연산자)는 나눗셈을 하고 남은 찌꺼기만 가져옵니다. 짝수/홀수를 판별할 때 아주 유용하죠!

// 1. 일반적인 나누기 (몫을 구함)
printf("%d", 10 / 3); // 출력: 3

// 2. 나머지 연산 (나머지를 구함)
printf("%d", 10 % 3); // 출력: 1

// 💡 꿀팁: 2로 나눈 나머지가 0이면 짝수, 1이면 홀수!
printf("%d", 4 % 2); // 출력: 0 (짝수)

이제 아래의 교과서 예제(86.c, 87.c)를 통해 변수가 섞인 진짜 수식을 만나봅시다!

예제 연산자 우선순위
86.c 
#include <stdio.h>
int main()
{
	int result = 4 + 3 * 5;
	printf("%d", result);
	return 0;
}
코드 상세 풀이

수학에서와 마찬가지로 C언어에서도 곱셈(*)이 덧셈(+)보다 먼저 계산됩니다.

  • 3 * 5가 먼저 계산되어 15가 됩니다.
  • 그 다음 4 + 15가 계산되어 19가 result 변수에 저장됩니다.
예제 5칙 연산 (나머지 포함)
87-1.c 
#include <stdio.h>
int main()
{
	int i = 13, j = 5;
	printf("%d %d %d %d %d", i + j, i - j, i*j, i / j, i%j);
	return 0;
}
코드 상세 풀이

i = 13, j = 5일 때 헷갈리기 쉬운 두 가지 연산을 확인해 봅시다.

  • i / j (나눗셈): 13을 5로 나누면 2.6이지만, 정수끼리의 나눗셈은 소수점 이하를 버리므로 2가 출력됩니다.
  • i % j (나머지): 13을 5로 나눈 후 남은 나머지인 3이 출력됩니다. (13 = 5 * 2 + 3)
예제 대입 연산
87-2.c 
#include <stdio.h>
int main()
{
	int i = 13, j = 5;
	i = j;
	printf("%d\n", i);
	return 0;
}
코드 상세 풀이

프로그래밍에서 대입 연산자(=)는 "같다"는 뜻이 아니라, "오른쪽의 값을 왼쪽 변수에 넣어라"는 의미입니다.

  • i = j; 코드는 "j의 값(5)을 복사해서 i에 덮어씌워라"라는 뜻입니다.
  • 이때 i가 원래 가지고 있던 13이라는 값은 지워지고, 새로운 값인 5가 됩니다.

증감 연산자 (++, --)

워밍업: 1씩 더하고 빼기 (단독 사용)

++는 현재 값에 1을 더하라는 뜻이고, --는 1을 빼라는 뜻입니다.
가장 중요한 사실은, 다른 명령어 없이 혼자 쓰일 때는 앞에 붙든(++a) 뒤에 붙든(a++) 결과가 완전히 똑같다는 것입니다.

int a = 10;
a++; // a는 11이 됨
printf("%d", a); // 11 출력
int b = 10;
++b; // b는 11이 됨
printf("%d", b); // 11 출력

🚨 그런데 왜 헷갈린다고 할까요?
아래 교과서 예제(88.c)처럼 temp = ++i; 와 같이 '대입 연산(=)'과 섞어 쓸 때 계산 순서가 달라지기 때문입니다. 아래 예제에서 그 차이를 확인해 보세요!

독학 포인트: 전위(++a)와 후위(a++)가 헷갈리시나요?

초보자들이 가장 많이 헷갈려 하는 부분입니다. 핵심은 "언제 1을 더할 것인가?"입니다. 식당에 비유해 볼까요?

++a (전위 연산)

"선불 식당"입니다. 밥을 먹기(문장 실행) 전에 먼저 1을 더합니다.

int a = 10;
int result = ++a;
// a가 11이 된 후, result에 11이 들어감
a++ (후위 연산)

"후불 식당"입니다. 현재 값을 사용해 밥을 다 먹은(문장 실행) 다음에 1을 더합니다.

int a = 10;
int result = a++;
// result에 10이 먼저 들어가고, 그 후 a가 11이 됨

아래의 복잡한 88.c 예제를 보기 전에, 이 원리를 꼭 기억하세요!

예제 전치 vs 후치 연산의 차이
88.c 
#include <stdio.h>
int main()
{
	int i = 13, j = 5, temp,
		a = 10, b = 20, c = 30, d = 40;
	temp = ++i; printf("%d %d\n", temp, i);
	temp = i++; printf("%d %d\n", temp, i);
	temp = --j; printf("%d %d\n", temp, j);
	temp = j--; printf("%d %d\n", temp, j);
	++a; b++; --c; d--;
	printf("%d %d %d %d", a, b, c, d);
	return 0;
}
코드 상세 풀이 (한 줄씩 따라가 보세요!)

초기값: i = 13, j = 5

  • temp = ++i; (전위) i를 1 증가시켜 14로 만듭니다. 그 후 temp에 14를 넣습니다. (출력: 14 14)
  • temp = i++; (후위) 현재 i값인 14를 temp에 먼저 넣습니다. 그 다음 i가 15로 증가합니다. (출력: 14 15)
  • temp = --j; (전위) j를 1 감소시켜 4로 만듭니다. 그 후 temp에 4를 넣습니다. (출력: 4 4)
  • temp = j--; (후위) 현재 j값인 4를 temp에 먼저 넣습니다. 그 다음 j가 3으로 감소합니다. (출력: 4 3)

3. 관계, 삼항 및 논리 연산자

워밍업: 컴퓨터는 '맞다/틀리다'를 어떻게 표현할까?

C언어는 "True(참)"나 "False(거짓)"라는 글자 대신 숫자로 정답을 말합니다.
질문이 맞으면 1 (참)을, 틀리면 0 (거짓)을 만들어 냅니다.

// 1. 당연히 맞는 질문 (참)
printf("%d", 10 > 5);
// 10이 5보다 크므로 1 출력
// 2. 어이없는 질문 (거짓)
printf("%d", 10 == 5);
// 10과 5는 같지 않으므로 0 출력

컴퓨터가 1과 0으로 대답한다는 것을 기억하며, 아래의 복잡한 조건식(89-1.c)들을 해석해 봅시다!

예제 관계 연산자
89-1.c 
#include <stdio.h>
int main()
{
	int i = 13, j = 5;
	printf("%d %d %d %d %d %d",
		ij, i <= j, i >= j, i != j, i == j);
	return 0;
}
코드 상세 풀이

관계 연산의 결과는 조건이 맞으면 1 (참), 틀리면 0 (거짓)입니다. (i=13, j=5)

  • i < j (13 < 5) : 틀렸으므로 0
  • i > j (13 > 5) : 맞으므로 1
  • i <= j (13 <= 5) : 틀렸으므로 0
  • i >= j (13 >= 5) : 맞으므로 1
  • i != j (13과 5는 다르다) : 맞으므로 1
  • i == j (13과 5는 같다) : 틀렸으므로 0

잠깐! 삼항 연산자( ? : )란 무엇일까요?

삼항 연산자는 조건에 따라 두 가지 값 중 하나를 선택하는 아주 간편한 연산자입니다. (조건식) ? 참일_때_값 : 거짓일_때_값 형태로 사용합니다. 마치 짧은 질문을 던지는 것과 같습니다.

나이 >= 20
?
"성인" (참)
:
"미성년자" (거짓)

아래 예제에서 i < j ? 10 : 100은 "i가 j보다 작니? 맞으면 10, 틀리면 100을 줄게!"라는 의미입니다.

예제 삼항 연산자
89-2.c 
#include <stdio.h>
int main()
{
	int i = 13, j = 5, result;
	result = i < j ? 10 : 100;
	printf("%d", result);
	return 0;
}
코드 상세 풀이
  • i < j ? 10 : 100 라는 식은 "13이 5보다 작아?" 하고 질문을 던집니다.
  • 13은 5보다 작지 않으므로 조건은 거짓(False)이 됩니다.
  • 결과가 거짓일 때는 콜론(:) 뒤에 있는 값인 100이 최종 선택되어 result 변수에 저장됩니다.
예제 논리 연산자
90-1.c 
#include <stdio.h>
int main()
{
	int i = 13, j = 5, temp;
	printf("%d %d\n", i == 13 && j == 5, i == 13 && j == 4);
	printf("%d %d\n", i == 13 || j == 4, i == 12 || j == 4);
	temp = i; printf("%d ", !temp);
	temp = 0; printf("%d", !temp);
	return 0;
}
코드 상세 풀이
  • i == 13 && j == 5 : 앞뒤 조건이 모두 참이므로 결과는 1 (AND 연산)
  • i == 13 && j == 4 : j가 4라는 조건이 거짓이므로 결과는 0
  • i == 13 || j == 4 : i가 13이라는 조건이 참이므로, 하나만 참이어도 되는 OR 연산 결과는 1
  • !temp (NOT 연산) : temp가 13(0이 아닌 수는 모두 참)일 때는 반대인 거짓(0)이 되고, temp가 0일 때는 반대인 참(1)이 출력됩니다.
예제 단락 회로 평가(Short-Circuit)
90-2.c 
#include <stdio.h>
int main()
{
	int i = 13, j = 5;
	if (i < 10 && ++j > 3) { } // i < 10이 거짓이므로 ++j는 실행되지 않음!
	printf("%d %d\n", i, j);   // 결과: 13 5
	return 0;
}
왜 j가 6이 아니라 5인가요?
A && B (AND 연산)

앞의 조건(A)이 거짓(0)이면, 뒤의 조건(B)은 결과와 상관없이 **아예 실행하지 않습니다.**

A || B (OR 연산)

앞의 조건(A)이 참(1)이면, 뒤의 조건(B)은 결과와 상관없이 **아예 실행하지 않습니다.**

i < 10 (F) && ++j > 3 결과: 거짓(0)

"이미 앞쪽에서 거짓으로 결론이 났기 때문에, 뒤쪽의 ++j는 아예 쳐다보지도 않습니다."

비트 연산자

비트(Bit) 연산자, 0과 1의 세계로!

비트 연산자는 컴퓨터가 데이터를 저장하는 가장 작은 단위인 0과 1(2진수)을 직접 다루는 고급 연산자입니다. 일상적인 프로그래밍보다는 하드웨어 제어나 알고리즘 최적화에 주로 쓰입니다.

1. 비트 논리 연산자
  • & AND: 둘 다 1일 때만 1
  • | OR: 하나라도 1이면 1
  • ^ XOR: 서로 다를 때만 1
  • ~ NOT: 0은 1로, 1은 0으로 뒤집기
2. 시프트(Shift) 연산자

비트를 왼쪽이나 오른쪽으로 밀어냅니다.

  • << (왼쪽 이동): 값이 2배씩 커집니다.
  • >> (오른쪽 이동): 값이 2로 나누어집니다.
예: 5 (0101) << 1 = 10 (1010)
예제 비트 논리 연산자 (&, |, ^, ~)
91-1.c 
#include <stdio.h>
int main()
{
	int i = 13;  // 00000000 00000000 00000000 00001101
	int j = 5;   // 00000000 00000000 00000000 00000101
	printf("%d %d %d %d", i&j, i | j, i^j, ~i);
	return 0;
}
예제 시프트 연산자 (<<, >>)
91-2.c 
#include <stdio.h>
int main()
{
	int i = 13;  // 00000000 00000000 00000000 00001101
	int j = 5;   // 00000000 00000000 00000000 00000101
	printf("%d %d %d %d", i << 2, i >> 2, j << 3, j >> 3);
	return 0;
}

연산자 응용 예제 종합 심화학습!

실전 응용으로 넘어가기 전 가이드

이제부터는 scanf로 직접 값을 입력받아 실제 생활에 쓰일 법한 계산기들을 만들어 봅니다. 아래 예제들의 수식이 길어 보인다고 당황하지 마세요!

  • 컴퓨터는 우리가 배운 우선순위(*, / 먼저, 그 다음 +, -)대로 차근차근 계산합니다.
  • 수식이 헷갈릴 때는 변수에 값을 직접 대입해서 손으로 한 번 계산해 보면 훨씬 이해가 빠릅니다.
예제 정기예금 이자 계산기
92.c 
#include <stdio.h>
int main()
{
	int money, duration;
	double rate;
	double result;
	printf("정기예금 금액을 입력해 주세요 :");
	scanf("%d", &money);
	printf("정기예금 이율을 입력해 주세요 :");
	scanf("%lf", &rate);
	printf("정기예금 기간을 입력해 주세요 :");
	scanf("%d", &duration);
	result = money + money * duration * 0.01 * rate;
	printf("%d원을 %.1f%%이율로 %d년간 예치 후 원리 합계는 %.0f입니다.",
		money, rate, duration, result);
	return 0;
}

왜 나누기 결과에 (float)를 붙일까요? (형변환)

아래 예제(93.c)에서 (float)(a / b)라는 낯선 문법이 등장합니다.
C언어에서는 정수와 정수를 나누면 결과도 무조건 정수(소수점 버림)가 됩니다. (예: 5 / 2 = 2)

❌ 그냥 나눌 때
5 / 2 ➔ 2 (0.5가 날아감)
✅ (float)를 붙일 때
(float)5 / 2 ➔ 2.500000

이렇게 데이터의 형태를 임시로 바꾸는 것을 '형변환(Casting)'이라고 합니다. 정확한 소수점 결과를 얻고 싶을 때 사용해요!

예제 사칙연산 계산
93.c 
#include <stdio.h>
int main()
{
	int a, b;
	printf("두 수를 입력하세요 :");
	scanf("%d %d", &a, &b);
	printf("%d + %d = %d\n", a, b, a + b);
	printf("%d - %d = %d\n", a, b, a - b);
	printf("%d * %d = %d\n", a, b, a * b);
	printf("%d / %d = %f\n", a, b, (float)(a / b));
	printf("%d %% %d = %d" , a, b, a % b);
	return 0;
}
예제 구속 변환 (마일 → km)
94-1.c 
#include <stdio.h>
int main()
{
	int mileSpeed;
	double kmSpeed;
	printf("볼의 속도 입력(마일) : ");
	scanf("%d", &mileSpeed);
	kmSpeed = mileSpeed * 1.609;
	printf("%d마일은 %.3fkm입니다.", mileSpeed, kmSpeed);
	return 0;
}

예제 더치페이 계산 1 (균등 분배)
95-1.c 
#include <stdio.h>

	printf("총 인원 : ");

코드를 더 우아하게: 나머지(%) 연산자의 마법

아래 95-2.c 예제에서 1명이 몰아서 내야 할 자투리 금액을 구하는 공식이 다소 복잡합니다.
price - (price / person) * (person - 1)
이 공식은 사실 "총 금액을 인원수로 나눈 나머지"와 "1명 몫"을 더한 것과 같습니다.

하지만 우리가 맨 처음에 배운 나머지 연산자(%)를 활용하면 공식을 훨씬 직관적이고 아름답게 바꿀 수 있습니다!

// 1명이 내야 할 총 금액 (기본 몫 + 남은 자투리 금액)
(price / person) + (price % person)

아래의 복잡한 예제 코드 식과, % 연산자를 사용한 식을 마음속으로 비교해 보세요! 프로그래머에게 % 연산자는 아주 훌륭한 무기입니다.

예제 더치페이 계산 2 (나머지 몰아주기)
95-2.c 
#include <stdio.h>
int main()
{
	int price, person;
	printf("총 금액 : ");
	scanf("%d", &price);
	printf("총 인원 : ");
	scanf("%d", &person);
	printf("%d명이 각각 %d원씩 지불", person-1, price / person);
	printf(", 1명 %d원 지불", price - (price/person)*(person-1));
	return 0;
}
