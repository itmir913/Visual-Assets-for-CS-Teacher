Chapter 2. C 프로그래밍의 기초

4. 제어문 (Control Statement), 반복 구조

프로그램의 흐름을 내 마음대로! 조건에 따라 다른 길을 가거나, 똑같은 작업을 반복하는 제어 구조를 단계적으로 마스터합니다.

학습 목표 (Objective)

  • 순차, 선택, 반복의 제어 구조를 이해하고 예제로 직접 활용할 수 있다.
  • 중첩 제어 구조(제어문 안의 제어문)를 다양한 패턴에 적용할 수 있다.

순차 구조

위에서 아래로 정해진 순서에 따라 차례대로 코드가 실행되는 가장 기본적인 흐름 구조입니다.

선택 구조

주어진 조건의 참·거짓에 따라 서로 다른 코드를 실행하는 분기 구조입니다. (if, switch)

반복 구조

특정 조건이 만족될 때까지 동일한 코드를 계속 반복 실행하여 작업을 자동화합니다. (for, while)

반복문 (Iteration) 의 마법

동일한 작업을 수천, 수만 번 직접 타이핑할 필요가 없습니다. 반복문은 정해진 횟수나 특정 조건이 만족될 때까지 컴퓨터가 대신 코드를 반복해서 실행해 주는 프로그래머의 가장 강력한 무기입니다.

반복문이 없다면? (코드 100번 쓰기)

1printf("Hello\\n");
2printf("Hello\\n");
3printf("Hello\\n");
97번 더 직접 작성해야 함... 😭

반복문을 쓴다면? (단 3줄!)

1for (int i = 0; i < 100; i++) {
2printf("Hello\\n");
3}
생산성 10000% 상승

for 문

반복 횟수가 명확할 때 사용 (예: 10번 반복)

횟수 중심

for문은 어떤 순서로 움직일까?

for ( ① 초기식 ; ② 조건식 ; ④ 증감식 ) {
    ③ 실행문 ;
}
  • 1

    초기식: 반복을 시작할 때 처음 딱 한 번만 실행 (i = 1)

  • 2

    조건식: 참이면 진입, 거짓이면 즉시 종료 (i <= 10)

  • 3

    실행문: 조건이 참일 때 실행할 메인 코드

  • 4

    증감식: 실행문이 끝나면 위로 올라와 변수 업데이트 (i++) 후 다시 ②번 조건식으로!

기초 예제 for_basic.c 
int sum = 0;
// for(초기식; 조건식; 증감식)
for (int i = 1; i <= 10; i++) {
    sum = sum + i;
}
printf("1~10 합: %d", sum);
응용 예제 103.c (구구단 2단 출력) 
#include <stdio.h>

int main() {
    int i;
    // i가 1부터 9까지 1씩 증가하며 반복
    for (i = 1; i < 10; i++) {
        printf("2 x %d = %d\n", i, 2 * i);
    }
    return 0;
}

학생 단골 실수: for문 끝에 세미콜론(;)을 붙이지 마세요!

조건식 끝에 무심코 ;을 붙이면, 컴퓨터는 '아무것도 안 하고' 5번 반복이 끝난 것으로 인식합니다. 그 결과 아래 중괄호 안의 코드는 단 1번만 실행되는 치명적인 버그가 발생합니다.

for (int i = 1; i <= 5; i++); // ← 치명적 실수!
{
    printf("안녕하세요\n");
}

while 문

특정 조건이 유지되는 동안 계속 반복 (예: 게임이 끝날 때까지)

조건 중심

while문은 어떤 순서로 움직일까?

while ( ① 조건식 ) {
    ② 실행문 ;
}
  • 1

    조건식: 반복을 시작하기 전, 매번 먼저 확인합니다. 참이면 진입, 거짓이면 즉시 종료.

  • 2

    실행문: 조건이 참일 때 실행됩니다. 실행이 끝나면 다시 ① 조건식으로 돌아가 참인지 또 확인합니다.

기초 예제 while_basic.c 
int i = 1, sum = 0;
// 조건식이 참인 동안 반복
while (i <= 10) {
    sum += i;
    i++; // 증감식 필수!
}
printf("1~10 합: %d", sum);
응용 예제 104-1.c (0 입력시 종료) 
#include <stdio.h>

int main() {
    int answer = 1;

    // 0이 입력될 때까지 계속 반복
    while (answer != 0) {
        printf("숫자를 입력하세요 (0을 누르면 종료) : ");
        scanf("%d", &answer);
        printf("%d 입력\n", answer);
    }
    return 0;
}

무한 루프(Infinite Loop) 주의!

while문 블록 안에 조건을 변화시키는 코드(예: i++)를 빼먹으면, 조건이 영원히 참이 되어 프로그램이 멈추지 않고 영원히 돌게 됩니다.

int i = 0;
while (i < 5) {
    printf("안녕하세요\n");
    // i++; (이 코드가 빠지면 i는 계속 0이므로 무한 반복!)
}

반복 구조 (do-while 문)

do-while 문은 기본적으로 while 문과 비슷하지만, 조건의 참·거짓과 상관없이 먼저 한 번은 실행한 뒤에 조건을 검사한다는 점이 다릅니다.

독학 포인트: do-while문은 어떤 순서로 움직일까?

do {
    ① 실행문 ;
} while ( ② 조건식 );
  • 1

    실행문: 반복문 안의 코드를 무조건 한 번 실행합니다. 조건문을 판별하지 않고, 중괄호 { } 안의 내용을 먼저 실행됩니다.

  • 2

    조건식: 실행문이 끝난 후 확인합니다. 참이면 다시 실행문으로 돌아가 반복하고, 거짓이면 종료합니다. (예: i < 10)

핵심 통찰: while과 do-while은 언제 다를까?

앞서 배운 104-1.c (while문) 코드를 보면 int answer = 1;처럼 시작부터 억지로 1을 넣어주었습니다. 만약 0을 넣으면 시작하자마자 조건(answer != 0)이 거짓이 되어 입력을 단 한 번도 받지 못하고 프로그램이 끝나버리기 때문이죠.

하지만 do-while문은 일단 중괄호 안의 코드를 무조건 1번은 실행합니다. 따라서 초기값을 억지로 세팅할 필요 없이 사용자에게 "일단 무언가를 입력받고 나서 검사해야 하는 상황"에서 훨씬 자연스럽고 깔끔한 코드가 됩니다. (아래 105-1.c 예제를 보세요!)

예제 do ~ while문 (조건 확인 나중)
105-1.c 
#include <stdio.h>
int main()
{
    int answer;
    do {
        scanf("%d", &answer);
        printf("%d 입력\n", answer);
    } while (answer != 0);
    return 0;
}

1회차 (무조건 실행) 5 입력 5 입력\n 참 (5 != 0) -> 반복
2회차 0 입력 0 입력\n 거짓 (0 != 0) -> 종료

#include <stdio.h>
int main()

2회차 소문자 'q' 대문자 'Q' 출력 거짓 -> 반복 종료

잠깐! 문자에 어떻게 숫자를 빼나요? (ch - 32)

컴퓨터는 문자를 저장할 때 고유의 숫자(아스키코드)로 바꾸어 저장합니다. 소문자 'a'는 97이고, 대문자 'A'는 65입니다. 소문자와 대문자는 항상 32만큼 차이가 나기 때문에, 소문자에서 32를 빼면 대문자가 되는 마법이 일어납니다! 반복문과 함께 자주 쓰이는 C언어의 특징입니다.

반복 흐름 제어문 (Iteration Control Statement)

반복문을 실행하다가 강제로 빠져나오거나, 특정 부분만 건너뛰고 싶을 때 break와 continue를 사용합니다.

break (즉시 탈출)

조건을 만족하면 반복문을 그 즉시 완전히 종료합니다. 

for (int i = 1; i <= 10; i++) {
    if (i == 5) break;
    printf("%d ", i);
}
// 결과: 1 2 3 4

continue (건너뛰기)

이어지는 명령문을 무시하고 다음 회차로 넘어갑니다. 

for (int i = 1; i <= 5; i++) {
    if (i == 3) continue;
    printf("%d ", i);
}
// 결과: 1 2 4 5

반복 흐름 제어문 (break 문)

break 문과 continue 문은 조건문이나 반복문의 실행 흐름을 제어하는 명령문입니다.

앞선 선택 구조 단원에서 "C언어는 0이 아닌 모든 수를 참(True)으로 본다"고 배웠습니다. 따라서 while (1)이라고 적으면 조건이 "무조건 참"이 되어 의도적인 무한 루프가 만들어집니다.

무한 루프를 만들 때는 프로그램이 먹통이 되지 않도록 반드시 내부에 if 조건문과 break;를 짝꿍처럼 결합하여 "탈출구"를 만들어 주어야 합니다.

예제 break 문
106.c 
#include <stdio.h>
int main()
{
    int answer;
    while (1)
    {
        scanf("%d", &answer);
        if (answer == 0)
        {
            break;
        }
        else {
            printf("%d 입력", answer);
        }
    }
    return 0;
}
👀 눈으로 보는 반복문: 106.c 변수 추적표 보기 (클릭)

💡 가정: 사용자가 10, 20, 0을 순서대로 입력했다고 가정합니다. (초기 sum = 0)

반복 회차 num 입력 현재 sum 누적값 탈출 검사 (if num == 0)
1회차 10 0 + 10 = 10 거짓 (계속 진행)
2회차 20 10 + 20 = 30 거짓 (계속 진행)
3회차 0 30 + 0 = 30 참 -> break (즉시 탈출!)

반복 흐름 제어문 (continue 문)

continue 문은 현재 반복을 건너뛰고 다음 반복으로 넘어가도록 흐름을 제어하는 명령문입니다.

예제 for문과 continue문 (107-1.c)
107-1.c 
#include <stdio.h>
int main()
{
    int i, count = 0, answer;
    scanf("%d", &answer);
    for (i = 1; i < 101; i++)
    {
        if (i % answer != 0)
        {
            continue;
        }
        count++;
    }
    printf("1부터 100까지 수 중에서 %d의 배수는 %d개입니다.",
        answer, count);
    return 0;
}
👀 눈으로 보는 반복문: 107-1.c 변수 추적표 보기 (클릭)

💡 가정: 사용자가 answer로 3을 입력했다고 가정합니다. (1~5까지만 추적)

i 값 배수 확인 (i % 3 != 0) 실행 흐름 count 값 (초기 0)
1 참 (배수 아님) continue (건너뜀) 0
2 참 (배수 아님) continue (건너뜀) 0
3 거짓 (3의 배수!) count++ 실행 1
4 참 (배수 아님) continue (건너뜀) 1

나머지 연산자(%) 활용 팁: 배수 판별법

조건식의 i % answer != 0은 변수 i를 answer로 나누었을 때 나머지가 0이 아니라는 뜻입니다. 나누어 떨어지지 않는다는 것은 곧 "배수가 아닐 때"를 의미하죠. 즉, 배수가 아니면 continue;를 만나 밑으로 가지 않고 건너뛰게 되므로, 결과적으로 배수일 때만 count++가 실행되어 배수의 개수를 셀 수 있습니다.

예제 while문과 continue문 (107-2.c)
107-2.c 
#include <stdio.h>
int main()
{
    int i = 0;
    while (i++ < 10) {
        if (i == 7) continue;
        printf("%d ", i);
    }
    return 0;
}

주의! while(i++ < 10) 의 숨겨진 작동 원리

조건 검사를 할 때 i < 10을 먼저 확인하고, 그 즉시 i가 1 증가한 채로 블록 안으로 들어갑니다! 즉 i가 1일 때 조건은 참이지만, 블록 안의 코드를 실행할 때 i는 이미 2가 되어 있습니다. 이 미세한 차이를 아래 표에서 꼭 확인하세요.

초보자 단골 실수: while문에서 continue를 쓸 때 조심하세요!

for문과 달리 while문은 증감식(i++)을 보통 블록 맨 아래에 적습니다. 만약 증감식 위에 continue;가 있으면 어떻게 될까요? 증감식을 만나기도 전에 위로 튕겨 올라가 버리기 때문에, 변수 값이 영원히 변하지 않고 무한 루프에 빠지게 됩니다.

이를 방지하기 위해 107-2.c 예제에서는 아예 조건식 안에 while (i++ < 10)처럼 조건 검사와 증감을 하나로 합쳐서 작성했습니다. 아주 영리한 코드죠!

다음 시간 맛보기: 중첩 반복문 (Nested Loop)

반복문 안에 또 다른 반복문을 넣는 중첩 제어 구조입니다. 논리적 사고가 필요해 초보자들이 가장 어려워하는 구간이지만, '시계바늘'을 떠올리면 이해하기 쉽습니다.

시계의 분침과 초침의 원리

시계의 분침(바깥쪽 반복문)이 딱 1칸 움직이려면, 초침(안쪽 반복문)은 60칸을 팽그르르 돌아야 합니다. 바깥쪽 반복문이 1번 실행될 때마다, 안쪽 반복문은 자신의 처음부터 끝까지 완전히 다 돌아야만 바깥쪽이 다음 회차로 넘어갑니다.

예제 중첩 for문 (직각 삼각형 별 찍기)
nested_for.c 
#include <stdio.h>

int main() {
    // 바깥쪽 루프 (줄바꿈 역할, 분침)
    for (int i = 1; i <= 3; i++) {

        // 안쪽 루프 (별을 옆으로 찍는 역할, 초침)
        for (int j = 1; j <= i; j++) {
            printf("*");
        }
        printf("\n"); // 안쪽 루프가 끝나면 한 줄 띄우기
    }
    return 0;
}

반복 구조 핵심 요약

  • 용도에 따른 반복문 선택 횟수가 정해진 반복은 for 문을, 특정 조건이 유지되는 동안의 반복은 while 문을 주로 사용합니다.
  • 최소 1회 실행 보장 (do-while) do-while 문은 조건을 나중에 검사하므로, 조건이 처음부터 거짓이더라도 무조건 본문을 한 번은 실행합니다.
  • 흐름 제어 (break와 continue) break는 반복문을 즉시 탈출하고, continue는 현재 회차를 건너뛰고 다음 반복(증감식)으로 이동합니다.
