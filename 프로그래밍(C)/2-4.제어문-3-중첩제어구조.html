4. 중첩 제어 구조 (Nested Control)

프로그램의 흐름을 내 마음대로! 제어문 안에 또 다른 제어문이 들어가는 복합 논리 구조를 단계적으로 마스터합니다.

학습 목표 (Objective)

  • 중첩 반복문의 작동 원리(안쪽 루프가 다 돌아야 바깥쪽 루프가 1보 전진)를 이해한다.
  • 반복 횟수 수식화, 변수 초기화 위치, 루프 탈출 조건 등 실전 알고리즘 필수 패턴 3가지를 익힌다.

순차 구조

위에서 아래로 차례대로 코드가 실행되는 가장 기본적인 흐름입니다.

선택 구조

조건의 참·거짓에 따라 서로 다른 코드를 실행합니다. (if, switch)

반복 구조

특정 조건이 만족될 때까지 동일한 코드를 계속 반복 실행합니다. (for, while)

개념 다지기: 중첩 반복문은 어떻게 돌아갈까?

반복문 안에 또 반복문이 들어가면 헷갈리기 시작합니다. 하지만 시계의 분침과 초침을 떠올리면 쉽습니다! 바깥쪽 루프(분침)가 1칸 움직일 때, 안쪽 루프(초침)는 정해진 횟수를 모두 돌아야 비로소 끝이 나고 분침이 다음 칸으로 넘어갑니다.

3x4 사각형 별 출력 

#include <stdio.h>
int main() {
    int i, j;
    // 바깥쪽 루프: 행(줄)을 담당
    for (i = 0; i < 3; i++) {
        // 안쪽 루프: 열(별의 개수)을 담당
        for (j = 0; j < 4; j++) {
            printf("* ");
        }
        printf("\n"); // 안쪽 4번이 다 끝나야 줄바꿈!
    }
    return 0;
}

한눈에 보는 실행 흐름

바깥쪽 (i) 안쪽 (j) 진행 과정 화면 출력
i = 0 j = 0 → 1 → 2 → 3 * * * * \n
i = 1 j = 0 → 1 → 2 → 3 * * * * \n
i = 2 j = 0 → 1 → 2 → 3 * * * * \n

실전 투입 전 베이스캠프: 필수 패턴 3대장

이제 아래의 3가지 패턴만 이해하면 앞으로 나올 모든 복잡한 알고리즘(도형, 소수, 완전수)을 직접 해석하고 짤 수 있습니다.

1. 동적 반복 (도형 그리기)

첫 줄엔 1번, 두 번째 줄엔 2번... 횟수가 변해야 한다면 안쪽 조건식에 바깥 변수(i)를 넣습니다. 
for (i = 1; i <= 3; i++) {
    for (j = 1; j <= i; j++) {
        // j는 i번까지만 돈다!
        printf("*");
    }
}

2. 변수 0 초기화 위치

매 줄마다, 혹은 매 숫자마다 합계를 새로 구해야 한다면 바깥 루프 바로 안쪽에서 0으로 리셋해야 합니다. 
for(i = 1; i <= 3; i++) {
    int sum = 0; // 💡 여기서 리셋!
    for(j = 1; j <= 3; j++) {
        sum += j;
    }
}

3. 루프 무사 탈출 판별

루프가 break로 쫓겨난 건지, 끝까지 돌았는지 알려면 반복 변수의 최종값을 검사하면 됩니다. 
for(i=2; i < num; i++){
    if(num % i == 0) break;
}
// i가 num까지 도달했으면 소수!
if(i == num) { ... }

실전 예제 1: 중첩 반복문과 도형

숫자 직각삼각형 (111.c)

111.c

[패턴 1] j <= i를 사용하여 줄이 바뀔수록 출력 횟수를 늘립니다.
Point! count는 아예 바깥쪽에 선언되어 0으로 리셋되지 않고 1부터 계속 누적 증가합니다. 

#include <stdio.h>
int main() {
    int i, j, count = 1; // 💡 밖에서 한 번만 초기화

    for (i = 1; i <= 5; i++) {
        for (j = 1; j <= i; j++) {
            printf("%d ", count++); // 출력 후 1 증가
        }
        printf("\n");
    }
    return 0;
}

공백 포함 트리 출력 (102.c)

102.c

하나의 바깥 루프 안에 2개의 안쪽 루프(공백용 j, 문자용 k)가 나란히 존재하는 구조입니다. 아래의 수식 변환표를 보면 원리가 쉽게 이해됩니다.

i (줄 번호) 공백 (5 - i) 문자 k <= (2 * i)
i = 0 5개 0~0 (1개)
i = 1 4개 0~2 (3개)
i = 2 3개 0~4 (5개) 
#include <stdio.h>
int main() {
    int i, j, k;
    for (i = 0; i < 5; i++) {
        // 1. 공백 출력 (줄어듦)
        for (j = 0; j < 5 - i; j++) putchar(' ');

        // 2. @ 출력 (홀수 개수로 늘어남)
        for (k = 0; k <= 2 * i; k++) putchar('@');

        puts(""); // 3. 줄바꿈
    }
    return 0;
}

실전 예제 2: 논리 알고리즘

소수(Prime Number) 판별

앞서 배운 [패턴 3]이 완벽하게 적용되는 예제입니다. `if (i == num)`의 의미를 유심히 살펴보세요.

109-2.c
소수가 아닌 경우

중간에 나누어 떨어지면 break 작동. 루프가 조기 종료되어 i는 num에 도달하지 못함.

소수인 경우

끝까지 나누어 떨어지지 않으면 i가 계속 증가하여 결국 i == num이 된 상태로 루프 종료.

#include <stdio.h>

int main() {
    int num, i;
    printf("숫자 입력: ");
    scanf("%d", &num);

    for (i = 2; i < num; i++) {
        // 나누어 떨어지면 소수가 아님! 즉시 탈출
        if (num % i == 0) break;
    }

    // 탈출 후 i값을 검사하여 원인 파악 (패턴 3)
    if (i == num) {
        printf("%d는 소수입니다.\n", num);
    } else {
        printf("%d는 소수가 아닙니다.\n", num);
    }
    return 0;
}
완전수(Perfect Number) 찾기

자신을 제외한 약수의 합이 자신과 같은 수. 앞서 배운 [패턴 2] 초기화 위치가 완벽히 적용됩니다.

110.c
코드 핵심 분석
  • 1. 초기화 (패턴 2): 검사할 숫자(i)가 바뀔 때마다 약수의 합(sum)을 0으로 리셋!
  • 2. 안쪽 루프 (j): 1부터 i-1까지 나누어보며 약수를 구해 sum에 누적 
#include <stdio.h>

int main() {
    int i, j, sum = 0, count = 0;

    // 1부터 10000까지 모든 숫자를 검사
    for (i = 1; i <= 10000; i++) {
        sum = 0; // 💡 핵심: 매 숫자마다 합계 0으로 리셋!

        // 약수를 구해 sum에 누적
        for (j = 1; j < i; j++) {
            if (i % j == 0) {
                sum += j;
            }
        }

        // 약수의 합이 자기 자신과 같으면 완전수
        if (i == sum) {
            printf("%d번째 완전수: %d\n", ++count, i);
        }
    }
    return 0;
}
