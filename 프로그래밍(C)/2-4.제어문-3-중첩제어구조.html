Chapter 2. C 프로그래밍의 기초

4. 제어문 (Control Statement), 중첩 제어구조

프로그램의 흐름을 더욱 정교하게! 제어문 안에 또 다른 제어문을 넣는 중첩 구조를 통해, 복잡한 논리와 패턴을 단계적으로 마스터합니다.

학습 목표 (Objective)

  • 중첩 반복문의 작동 원리(외부 반복문 1회 실행 시 내부 반복문이 전체 실행됨)를 완벽히 이해한다.
  • 반복 구조와 선택 구조를 결합하여 데이터를 원하는 조건에 맞게 걸러내는 논리를 익힌다.

1. 순차 구조

위에서 아래로 정해진 순서에 따라 코드가 실행됩니다.

2. 선택 구조

조건(if/switch)에 따라 서로 다른 코드를 실행합니다.

3. 반복 구조

조건이 만족될 때까지 코드(for/while)를 계속 반복합니다.

제어문 안의 제어문: 중첩 (Nesting)

이전 단원들에서 배운 선택 구조(if)와 반복 구조(for)는 단독으로 쓰일 때도 강력하지만, 서로 결합할 때 비로소 진정한 마법이 시작됩니다. 반복문 안에 또 다른 반복문이 들어가거나, 반복문 안에 선택문이 들어가는 것을 중첩 제어 구조(Nested Control Structure)라고 부릅니다.

시계열 비유: 시침과 분침

중첩 반복문을 이해하는 가장 쉬운 방법은 아날로그 시계를 떠올리는 것입니다.
바깥쪽 반복문이 '시(Hour)'라면, 안쪽 반복문은 '분(Minute)'입니다.
시침(바깥 반복문)이 1시간 움직이기 위해서는, 분침(안쪽 반복문)이 0분부터 59분까지 전부 돌아야 합니다.

잠깐! 시작하기 전에: 너무 어렵게 느껴지나요?

코드를 보기 전에, 우리가 학교 체육 시간에 흔히 겪었던 상황을 상상해 볼까요? 중첩 반복문은 사실 아주 단순한 지시사항일 뿐입니다.

체육 선생님의 지시

"모두 운동장 3바퀴 돌아!

단, 1바퀴 돌 때마다 멈춰서
팔벌려뛰기 5번씩 하고 다시 출발해!"

우리의 행동 (컴퓨터의 행동)

  • 1바퀴째 팔벌려뛰기 1, 2, 3, 4, 5 번 수행
  • 2바퀴째 팔벌려뛰기 1, 2, 3, 4, 5 번 수행
  • 3바퀴째 팔벌려뛰기 1, 2, 3, 4, 5 번 수행
총 팔벌려뛰기 횟수 = 15번 (3 × 5)

이것이 전부입니다! "운동장 도는 것(외부 반복문)" 안에 "팔벌려뛰기(내부 반복문)"가 들어있는 형태. 이게 바로 중첩 반복문입니다. 자, 이제 이 느낌을 머릿속에 기억한 채로 실제 코드가 어떻게 생겼는지 구경하러 가볼까요?

현미경 관찰: 코드 1줄과 변수 상태표 1:1 매칭

코드를 눈으로만 읽으면 머릿속에서 숫자가 엉키기 쉽습니다. 코딩 고수들은 에러가 날 때 반드시 종이를 꺼내 '변수 상태표(Trace Table)'를 그립니다.
아래에 가장 단순한 형태의 중첩 반복문(외부 2번 × 내부 3번)을 준비했습니다. 왼쪽의 코드와 오른쪽의 표를 같은 색상끼리 번갈아 가며 확인해 보세요.

변수 상태표 그리는 3단계 공식

STEP 1. 기둥 세우기 바깥쪽 반복문 변수(i)와 안쪽 반복문 변수(j)를 표의 맨 위에 적습니다.
STEP 2. 내부 꽉 채우기 외부 변수 i가 하나 고정되어 있을 때, 내부 변수 j를 끝까지 1씩 늘려가며 빈칸을 적습니다.
STEP 3. 결과 기록하기 조건이 '참'일 때 화면에 무엇이 출력되는지, 언제 '거짓'이 되어 종료되는지 씁니다.
trace_test.c 
// 1. 외부 반복문 (파란색 i)
for (int i = 1; i <= 2; i++) {

    // 2. 내부 반복문 (보라색 j)
    for (int j = 1; j <= 3; j++) {

        // 3. 출력 (초록색)
        printf("i=%d, j=%d | ", i, j);
    }

    // 4. 내부 반복문이 끝나면 줄바꿈
    printf("\n");
}

관전 포인트: 오른쪽 표를 보면, i가 1일 때, j가 1, 2, 3까지 전부 돌고 나서야 비로소 i가 2로 넘어가는 것을 명확히 볼 수 있습니다!

외부
i <= 2		 내부
j <= 3		 조건 판별 & 실행 결과
i = 1
(1회차 시작) 		j = 1 참(O) 출력: i=1, j=1 |
j = 2 참(O) 출력: i=1, j=2 |
j = 3 참(O) 출력: i=1, j=3 |
j = 4 거짓(X) 내부 반복문 종료 ➜ 줄바꿈(\n) 실행
i = 2
(2회차 시작) 		j = 1 j는 다시 1로 초기화! 참(O) 출력: i=2, j=1 |
j = 2 참(O) 출력: i=2, j=2 |
j = 3 참(O) 출력: i=2, j=3 |
j = 4 거짓(X) 내부 반복문 종료 ➜ 줄바꿈(\n) 실행
i = 3 - 거짓(X) 조건(i<=2) 불만족 ➜ 외부 반복문 포함 모든 루프 최종 종료!

한눈에 보는 중첩 반복문 실행 흐름

아래 코드는 외부 반복문(i)이 3번, 내부 반복문(j)이 2번 도는 중첩 for문의 예시입니다. 컴퓨터가 실제로 어떤 순서로 코드를 실행하는지 표를 통해 확인해 보세요.

nested_loop.c 
// 외부 반복문 (시침)
for (int i = 2; i <= 7; i+=2) {
    // 내부 반복문 (분침)
    for (int j = 1; j <= 2; j++) {
        printf("i=%d, j=%d\n", i, j);
    }
}

핵심 포인트: 외부 반복문이 한 번 실행될 때마다 내부 반복문은 처음부터 끝까지(새로 초기화되어) 완전히 실행됩니다. 따라서 printf()문의 총 실행 횟수는 외부 반복 횟수(3) × 내부 반복 횟수(2) = 6번이 됩니다.

외부
i <= 7
(2씩 증가: i+=2) 		내부
j <= 2 		조건 판별 & 실행 결과
i = 2
(1회차 시작) 		j = 1 참(O) 출력: i=2, j=1
j = 2 참(O) 출력: i=2, j=2
j = 3 거짓(X) 내부 반복문 종료 ➜ 줄바꿈(\n) 실행
i = 4
(2회차 시작) 		j = 1 j는 다시 1로 초기화! 참(O) 출력: i=4, j=1
j = 2 참(O) 출력: i=4, j=2
j = 3 거짓(X) 내부 반복문 종료 ➜ 줄바꿈(\n) 실행
i = 6
(3회차 시작) 		j = 1 j는 다시 1로 초기화! 참(O) 출력: i=6, j=1
j = 2 참(O) 출력: i=6, j=2
j = 3 거짓(X) 내부 종료 ➜ 줄바꿈(\n)
i = 8 - 거짓(X) 조건(i<=7) 불만족 ➜ 외부 반복문 포함 모든 루프 최종 종료!

반복문과 선택문의 만남 (Filtering)

반복문이 무작정 똑같은 코드를 여러 번 실행한다면, 선택문(if)을 반복문 안에 넣으면 원하는 조건의 데이터만 골라내는 마법을 부릴 수 있습니다. 1부터 10까지 반복하면서 짝수만 화면에 출력하는 예제를 살펴보겠습니다.

filter_even.c 
#include <stdio.h>

int main() {
    // 1부터 10까지 순차적으로 반복 (총 10회)
    for (int i = 1; i <= 10; i++) {
        // 내부에서 조건문(if)을 통해 i가 짝수일 때만 출력
        if (i % 2 == 0) {
            printf("%d ", i);
        }
    }
    printf("\n");
    return 0;
}

실행 결과

2 4 6 8 10

프로그래머의 통과의례: 별 찍기

중첩 반복문을 배울 때 전 세계 모든 학생들이 거쳐 가는 예제가 있습니다. 바로 화면에 별(*)로 도형을 그리는 것입니다. 외부 반복문은 '줄(Row)'을 바꾸고, 내부 반복문은 '칸(Column)'에 별을 출력합니다.

star_triangle.c 
#include <stdio.h>

int main() {
    int n = 5; // 5줄을 출력할 예정

    for (int i = 1; i <= n; i++) {       // 줄(행)을 담당하는 외부 반복문
        for (int j = 1; j <= i; j++) {   // 별의 개수를 담당하는 내부 반복문
            printf("*"); // 줄바꿈 없이 이어서 출력
        }
        printf("\n"); // 내부 반복문이 끝나면 1줄 띄우기!
    }
    return 0;
}

  • i=1 내부 반복문 조건(j <= 1): 별 1개 찍고 줄바꿈
  • i=2 내부 반복문 조건(j <= 2): 별 2개 찍고 줄바꿈
  • ... 외부 반복문 변수 i가 내부 반복문의 종료 조건으로 쓰이는 마법입니다!

실생활 대표 예제: 구구단 전체 출력

중첩 반복문을 가장 직관적으로 이해할 수 있는 또 다른 예제는 구구단입니다.
'단(2단~9단)'이 한 번 바뀔 때마다, '곱하는 수(1~9)'는 처음부터 끝까지 전부 돌아야 완성된 구구단이 나옵니다.

gugudan.c 
#include <stdio.h>

int main() {
    // 외부 반복문: 2단부터 9단까지
    for (int dan = 2; dan <= 9; dan++) {
        printf("--- %d단 ---\n", dan);

        // 내부 반복문: 1부터 9까지 곱하기
        for (int i = 1; i <= 9; i++) {
            printf("%d x %d = %d\n", dan, i, dan * i);
        }
        printf("\n"); // 한 단이 끝나면 줄바꿈
    }
    return 0;
}

💡 실행 흐름 포인트

  • 1. 외부 반복문 시작 (dan = 2) --- 2단 ---이 먼저 출력됩니다.
  • 2. 내부 반복문 전체 실행 (i = 1 ~ 9) 2x1=2 부터 2x9=18 까지 9번의 출력이 연달아 발생합니다. 내부 반복문이 끝나야만 비로소 외부 반복문의 다음 단계로 넘어갈 수 있습니다.
  • 3. 외부 반복문 증가 (dan = 3) 이제 3단이 시작되고, 내부 반복문 i는 다시 1부터 새롭게 시작됩니다.

개념 미싱링크: 중첩 반복문에서 break를 만나면?

반복문을 강제로 탈출하는 break 문. 그렇다면 반복문이 두 개 겹쳐 있는 중첩 제어 구조에서 break를 사용하면 두 개의 반복문을 한 번에 모두 빠져나갈까요? 정답은 "아니오"입니다.

절대 규칙

break 문은 오직 자신이 속해 있는 가장 안쪽의 반복문 하나만 탈출합니다. 바깥쪽 반복문은 영향을 받지 않고 계속 돌아갑니다. 

for (int i = 1; i <= 3; i++) {
    printf("외부 반복문 %d회차 시작\n", i);

    for (int j = 1; j <= 5; j++) {
        if (j == 3) {
            printf("  -> j가 3이 되어 내부 반복문만 탈출!\n");
            break; // 내부 반복문(j)만 강제 종료됨. 외부(i)는 안 꺼짐!
        }
        printf("  내부 반복문 j = %d\n", j);
    }
}

내부와 외부의 변수명 충돌

외부 반복문에서 i를 썼는데, 내부에서도 무심코 i를 또 사용하면 논리가 완전히 엉키게 됩니다.

for (int i = 0; i < 5; i++) {
  // 안에서 또 i를 쓰면 무한루프 위험!
  for (int i = 0; i < 3; i++) { ... }
}

변수는 명확히 분리하기

관례적으로 외부부터 안쪽으로 i, j, k 순서로 변수명을 지정하여 헷갈리지 않도록 작성합니다.

for (int i = 0; i < 5; i++) {
  // 올바른 변수 j 사용
  for (int j = 0; j < 3; j++) { ... }
}

Mission 1

역삼각형 별 찍기

기존 별 찍기 예제를 수정하여, 5개부터 1개까지 거꾸로 줄어드는 역삼각형을 출력해 보세요.
💡 힌트: 처음부터 5줄로 시작해서 1줄로 줄어들게 하려면, 외부 반복문의 시작과 끝, 증감식(i--)을 어떻게 바꿔야 할까요?

*****
****
***

예제 코드 보기
solution_star.c 
#include <stdio.h>

int main() {
    // 외부 반복문을 5부터 1까지 1씩 감소(i--) 시킵니다.
    for (int i = 5; i >= 1; i--) {

        // 내부 반복문은 현재 줄 번호(i)만큼만 별을 찍습니다.
        for (int j = 1; j <= i; j++) {
            printf("*");
        }
        printf("\n");
    }
    return 0;
}
Mission 2

소수 (Prime Number) 찾기

2부터 50까지의 숫자 중에서 소수(1과 자기 자신으로만 나누어떨어지는 수)만 골라서 출력해 보세요. 반복문과 조건문의 완벽한 결합이 필요합니다.
💡 힌트: 나누어떨어지는 횟수(약수의 개수)를 세는 변수를 활용해 보세요. 약수가 딱 2개인 수가 소수입니다.

예제 코드 보기
solution_prime.c 
#include <stdio.h>

int main() {
    // 2부터 50까지의 숫자를 하나씩 검사합니다.
    for (int i = 2; i <= 50; i++) {
        int count = 0; // 약수의 개수를 셀 변수 초기화

        // 1부터 현재 숫자(i)까지 나누어 봅니다.
        for (int j = 1; j <= i; j++) {
            if (i % j == 0) { // 나누어 떨어지면
                count++;      // 약수 개수 1 증가
            }
        }

        // 약수가 1과 자기 자신, 딱 2개뿐이면 소수!
        if (count == 2) {
            printf("%d ", i);
        }
    }
    printf("\n");
    return 0;
}

중첩 제어구조 핵심 요약

  • 시계열의 원칙 (외부 1회 = 내부 전체) 외부 반복문이 1번 실행될 때, 내부 반복문은 처음부터 끝까지 새롭게 초기화되어 완벽히 1사이클을 회전합니다.
  • 독립적인 제어 변수 사용 논리적 오류와 무한 루프를 방지하기 위해, 외부와 내부 반복문은 반드시 서로 다른 제어 변수(예: i, j, k)를 사용해야 합니다.
  • 부분적 탈출 (Break의 한계) break 문은 자신이 속한 가장 안쪽의 반복문 하나만 탈출시킵니다. 중첩된 모든 반복문을 한 번에 종료하지 않습니다.
