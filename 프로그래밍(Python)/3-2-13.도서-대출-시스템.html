도서 대출 시스템 개발

딕셔너리 자료구조를 활용한 도서 정보 관리 및 대출/반납 프로세스 구현

Step 1. 프로그램 기획 (문제 정의)

도서관에서 책의 상태(대출 가능, 대출 중 등)를 관리하고, 사용자의 명령에 따라 대출과 반납 처리를 수행하는 프로그램을 기획합니다.

핵심 개념: 딕셔너리(Dictionary)

도서명을 Key로, 도서의 상태를 Value로 매핑하여 데이터를 관리합니다.

'파이썬' : '대출가능'
'C언어' : '대출중'
'자바' : '대출가능'

대출 처리

도서가 존재하고 '대출가능' 상태일 때만 '대출중'으로 변경합니다.

반납 처리

도서가 존재하고 '대출중' 상태일 때 '대출가능'으로 변경합니다.

Step 2. 프로그램 설계 (알고리즘)

명령어 입력에 따른 조건부 로직을 설계합니다.

명령어 구조 설계
  • 입력 형식: 명령어 도서명 (예: 1 파이썬)
  • 명령어 1: 대출 실행
  • 명령어 2: 반납 실행
  • 명령어 0: 프로그램 종료
예외 처리(Exception)

도서 목록에 없는 책을 요청할 경우 "존재하지 않는 도서" 메시지를 출력하도록 설계합니다.

상태 확인

이미 대출된 책을 다시 대출하려고 할 때의 거절 로직을 포함합니다.

Step 3. 프로그램 구현 (Python)

딕셔너리와 조건문을 활용한 핵심 코드 예시입니다.

# 도서 데이터 초기화
books = {"파이썬": "대출가능", "C언어": "대출가능", "자바": "대출가능"}

while True:
    cmd = input("명령 입력: ").split()
    if cmd[0] == '0': break

    op, title = cmd[0], cmd[1]

    if title not in books:
        print("존재하지 않는 도서입니다.")
    elif op == '1': # 대출
        if books[title] == "대출가능":
            books[title] = "대출중"
            print(f"{title} 대출 완료")
        else:
            print("이미 대출 중인 도서입니다.")
학습 포인트

books[title]을 통해 데이터에 접근할 때, 해당 title(Key)이 딕셔너리에 존재하는지 먼저 확인하는 것이 에러 방지의 핵심입니다.

Step 4. 프로그램 테스트 및 검토

실행 테스트 케이스

1. 대출 가능 도서 대출 시도
2. 대출 중인 도서 대출 시도
3. 없는 도서 요청
4. 반납 처리 후 다시 대출

온라인 채점 확인

다양한 입력값(Edge Case)에 대해서도 프로그램이 멈추지 않고 정상 작동하는지 확인하세요.
