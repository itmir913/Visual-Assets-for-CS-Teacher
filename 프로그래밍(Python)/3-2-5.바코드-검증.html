데이터 신뢰성 검사: 바코드 검증

수학적 알고리즘을 통해 입력된 바코드가 유효한지 판별하는 프로그램을 제작합니다.

바코드 끝자리의 비밀

우리가 마트에서 보는 바코드의 마지막 숫자는 아무렇게나 정해진 것이 아닙니다. 앞의 숫자들을 특정 공식으로 계산하여 나온 **체크섬(Checksum)**이죠. 이 숫자를 통해 바코드가 잘못 읽히지는 않았는지 스스로 검증할 수 있습니다.

Step 1. 프로그램 기획 (요구 사항)

8자리로 구성된 표준 바코드가 입력되었을 때, 마지막 자리가 올바른지 확인합니다.

8 8 0 1 2 3 4 5

(앞 7자리: 데이터 / 마지막 8번째: 체크 숫자)

입력 조건

8자리의 숫자가 공백 없이 문자열로 입력됩니다.
예: "88012345"

출력 조건

검증 결과가 맞으면 "Valid", 틀리면 "Invalid"를 출력합니다.

Step 2. 프로그램 설계 (검증 로직)

가장 널리 쓰이는 바코드 검증 공식(단순화 버전)을 적용해 봅시다.

체크 숫자 계산 단계
  1. 홀수 번째 자리에 있는 숫자들의 합을 구합니다. (\(S_1\))
  2. 짝수 번째 자리에 있는 숫자들의 합에 3을 곱합니다. (\(S_2\))
  3. \(S_1 + S_2\)를 계산하고, 이 합을 10으로 나눈 나머지(\(R\))를 구합니다.
  4. \(10 - R\)이 마지막 8번째 숫자와 일치하는지 확인합니다. (단, 결과가 10이면 0으로 처리)
데이터 처리 전략
  • 문자열로 입력받은 뒤 각 자리수를 숫자로 변환해야 합니다.
  • 반복문(for)을 사용하여 홀수/짝수 인덱스를 구분합니다.

Step 3. 프로그램 구현 (Source Code)

Python Solution
# 1. 바코드 입력 (문자열)
barcode = input()

# 2. 앞 7자리와 마지막 체크 숫자 분리
data = barcode[:7]
check_digit = int(barcode[7])

# 3. 합계 계산
s1 = 0 # 홀수 번째 합
s2 = 0 # 짝수 번째 합

for i in range(7):
    num = int(data[i])
    if (i + 1) % 2 == 1: # 인간 기준 홀수 번째
        s1 += num
    else:
        s2 += num

# 4. 검증 값 도출
total_sum = s1 + (s2 * 3)
remainder = total_sum % 10
result = (10 - remainder) % 10

# 5. 결과 출력
if result == check_digit:
    print("Valid")
else:
    print("Invalid")
프로그래밍 팁: (10 - remainder) % 10

계산 결과가 10일 때 0이 되어야 하는 조건은 나머지를 한 번 더 구하는 % 10 연산으로 우아하게 해결할 수 있습니다. 조건문을 줄이는 좋은 습관입니다!

Step 4. 프로그램 테스트 (검증)

작성한 프로그램이 다양한 바코드를 정확히 걸러내는지 확인해 봅시다.

정상 케이스

입력: 88012345
결과: Valid

오류 케이스

입력: 88012340
결과: Invalid

오류 발생 가능성 체크
  • 인덱스 번호 주의: 파이썬은 0부터 시작하지만, 공식은 '첫 번째(1번)'부터 따집니다. i+1 처리를 잊지 마세요.
  • 형 변환: 문자열인 data[i]를 그대로 더하면 오류가 납니다. 반드시 int()로 감싸주세요.
