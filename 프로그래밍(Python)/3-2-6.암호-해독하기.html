고대의 지혜: 시저 암호 해독하기

아스키코드와 반복문을 활용하여 암호화된 메시지를 원래대로 되돌리는 프로그램을 설계합니다.

시저 암호란?

로마의 장군 율리우스 시저가 사용했던 암호 방식입니다. 알파벳을 일정한 거리만큼 밀어서 글자를 바꾸는 방식이죠. 예를 들어 **3칸**을 밀기로 했다면, **'A'는 'D'**가 되고, 반대로 해독할 때는 **'D'에서 3을 빼서 'A'**를 찾습니다.

Step 1. 프로그램 기획 (요구 사항)

사용자가 입력한 암호문을 받아서, 정해진 규칙(왼쪽으로 3칸 이동)에 따라 해독한 결과를 출력합니다.

[해독 원리: 왼쪽으로 3칸 이동]

D
A

ASCII: 68 ('D') - 3 = 65 ('A')

입력 조건

알파벳 대문자로 구성된 암호문 문자열이 입력됩니다.
예: "DEFG"

출력 조건

각 글자를 왼쪽으로 3칸 밀어 해독한 원문을 출력합니다.
예: "ABCD"

Step 2. 프로그램 설계 (핵심 함수)

컴퓨터는 문자를 숫자로 인식합니다. 이 숫자가 바로 **아스키코드(ASCII)**입니다.

파이썬 필수 함수
  • ord(문자): 문자를 아스키코드 정수값으로 변환합니다.
  • chr(정수): 아스키코드 정수값을 다시 문자로 변환합니다.
알파벳 순환 처리 (Advanced)

만약 'A'에서 3을 빼면 어떻게 될까요? 알파벳 범위를 벗어나게 됩니다. 이번 프로젝트의 기본형은 범위를 벗어나지 않는 경우를 다루지만, 실제 구현 시에는 (x - 65 - 3) % 26 + 65와 같은 나머지 연산이 필요할 수 있습니다.

Step 3. 프로그램 구현 (Source Code)

Python Solution
# 1. 암호문 입력 받기
cipher_text = input()
plain_text = "" # 결과를 저장할 빈 문자열

# 2. 문자열의 각 문자를 하나씩 확인
for char in cipher_text:
    # 문자를 숫자로 변환 후 3을 뺌
    code = ord(char) - 3

    # 다시 문자로 변환하여 결과 문자열에 추가
    plain_text += chr(code)

# 3. 최종 해독 결과 출력
print(plain_text)
왜 문자열에 +=를 쓰나요?

문자열은 한 번에 바꿀 수 없기 때문에, 하나씩 해독한 글자를 새로운 바구니(plain_text)에 차곡차곡 쌓아가는 방식입니다.
